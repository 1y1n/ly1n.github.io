部署Hexo+github碰到的问题

Posted on 2017-02-17

#部署完成后，网页只显示文本，不显示样式，点击其他链接会出现404错误
按照网上的教程部署好所有需要的内容
打开本地调试 localhost:4000 显示正常
但用github项目设置中给的网址 https://xxx.github.io/xxx.github.io 进去只显示文本内容，所有网页样式都未显示
浏览器F12查看，显示js，css文件都是404。

在网上找资料未果，后面把仓库删了重新创建也不行。
后面在一个博客里看到说仓库名称一定要是 user_name.github.io 否则会出错。

重新创建仓库，使用ly-nina.github.io为名称，将本地博客内容deploy上去后，
进入Settings，发现Github Pages已经自动创建网页，心里想肯定这次有用了（因为前面我是自己点创建的）
果不其然，网页正常加载。

这里给自己提个醒，网上有些资料有坑，要多找资料对比。前面看的教程，没有明确表示Repository name一定且只能写自己github的名称，并且教程给的图片上名称就不一样。[捂脸]

Hexo+github博客简单部署大致如下

1、安装Git和Node.js(Windows下)
Node.js https://nodejs.org/en/
Git https://pan.baidu.com/s/1kU5OCOB#list/path=%2Fpub%2Fgit
（Git安装程序是在廖雪峰老师的网站上找到的，官网下载的实在是太慢了，附上Git教程）

2、Github帐号，并创建user_name.github.io同名仓库

3、在Git命令行里部署Hexo

hexo d 部署到Github之后就可以通过 https://ly-nina.github.io访问了，这里的ly-nina是我的github名称，你自己的具体网址可以在仓库Settings中查看

#中文显示乱码问题
写好博客文件之后，本地调试会发现写的中文都乱码了
处理方法：
把该xxx.md文件用记事本打开，点击另存为，在下面的编码选项中选择”UTF-8”，保存
重新部署，博客中文正常显示。
